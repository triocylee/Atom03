# Vampire Survivors 整体里程碑规划

## 项目目标
- 2天内复刻《Vampire Survivors》核心体验版
- PC平台，单人模式
- 俯视角自动射击生存玩法

## 核心玩法循环
1. 玩家移动 → 自动攻击范围内敌人
2. 击杀敌人 → 掉落XP宝石
3. XP积累 → 升级 → 选择新武器/强化现有武器
4. 存活时间越长 → 敌人生成越密集 → 最终死亡

## M1 Skeleton runnable（2天内必须完成）
### 阶段目标
- 打通最小可玩链路：玩家移动、自动攻击、敌人生成、XP升级

### 必须完成
1. 玩家控制：WASD移动，边界限制
2. 自动攻击系统：至少2种武器（近战+远程）
3. 敌人生成：时间驱动波次，敌人向玩家移动
4. XP系统：击杀掉落、拾取、等级提升
5. UI：HP、等级、武器状态、金币
6. 简单碰撞检测

### 验收标准
1. 从开局到死亡可完整跑通
2. 武器自动攻击正常工作
3. XP拾取和升级流程闭环
4. 同屏20+敌人流畅运行

### 角色推进顺序
1. Code先搭建项目骨架和核心系统
2. Art提供占位美术资源
3. QA编写冒烟测试用例

---

## M2 Core loop stable（后续迭代）
### 阶段目标
- 稳定核心玩法深度：更多武器、进化系统敌人差异化

### 必须完成
1. 武器数量扩展到4-6种
2. 武器进化路径（2-3条）
3. 3种以上敌人类型
4. 被动技能系统

### 验收标准
1. 多种策略可通关
2. 数值可配置
3. 帧率稳定

---

## M3 Content reinforcement（后续迭代）
### 阶段目标
- 增加可重复游玩内容：多地图、更多角色

### 必须完成
1. 2-3张可玩地图
2. 3-5个可玩角色
3. Boss机制

---

## M4 Polish and optimization（后续迭代）
### 阶段目标
- 提升手感、稳定性和表现力

### 必须完成
1. 战斗反馈完整（受击、击杀、屏幕震动）
2. UI/UX收口
3. 性能优化
4. 可交付试玩包

1. Art依赖Code的资源规格与挂点清单

4. 交接链：Code规格 → Art资源 → Code集成包 → QA测试报告 → Code修复
