# Thronefall 整体里程碑规划

## 项目目标
- 在单机 PC 端复刻《Thronefall》核心体验版："白天建设 + 夜晚守城"，包含可循环的经济、建造、战斗、失败/重开流程。

## 核心玩法循环
1. 白天：资源结算 -> 选择建设/升级 -> 调整防线。
2. 夜晚：敌军分波进攻 -> 玩家骑乘近战支援 + 防御塔输出 -> 王座血量判定。
3. 局后：胜负结算 -> 返回关卡准备下一轮。

## M1 Skeleton runnable
### 阶段目标
- 打通最小可玩链路：1 张地图、1 种资源、2 类建筑、1 类敌人、3 波夜袭。

### 必须完成
1. 地图与王座可放置、可寻路。
2. 白天建造与扣费逻辑可用。
3. 夜晚刷怪、塔攻击、王座受击与失败界面闭环。

### 验收标准
1. 从开局到失败/胜利可完整跑通，30 分钟无崩溃。
2. 建筑放置冲突、资源不足、王座归零均有正确反馈。
3. QA 能按脚本完成 20 次连续 smoke。

### 角色推进顺序
1. Code 先交互框架与占位数据。
2. Art 产出灰盒资源与占位特效。
3. QA 先做冒烟，再提交首轮缺陷。

### 协作与交接
- Code 每日打包可运行构建给 QA；Art 每批资源先给 Code 集成后再给 QA 回归。

## M2 Core loop stable
### 阶段目标
- 稳定核心策略深度：经济分支、建筑克制、敌人差异化。

### 必须完成
1. 4-5 种建筑功能分化（输出/减速/拦截/增益）。
2. 3 类敌人（步兵/高速/高血）与明确克制关系。
3. 白天决策对夜晚结果有统计可视化反馈。

### 验收标准
1. 至少 2 条可行策略可通关同一关卡。
2. 关键数值可配置并热更新。
3. 平均帧率满足目标机型下限。

### 角色推进顺序
1. Code 完成系统参数化与数据驱动。
2. Art 补齐建筑/敌人辨识度与命中特效。
3. QA 做平衡性与性能首轮回归。

### 协作与交接
- Code 先冻结接口；Art 按接口交付；QA 基于冻结版本执行对比测试。

## M3 Content reinforcement
### 阶段目标
- 增加可重复游玩内容：多关卡、多事件、多升级路径。

### 必须完成
1. 3 张可玩地图（不同地形与入口）。
2. 关卡内外升级选项与风险收益选择。
3. 失败后快速重开与关卡目标提示完善。

### 验收标准
1. 新玩家可在 10 分钟内理解并完成首关。
2. 每张地图至少有 1 个独特策略点。
3. 无阻断级 BUG（卡死、无法结算、存档损坏）。

### 角色推进顺序
1. Art 先地图灰盒与可读性。
2. Code 接入关卡流程与升级系统。
3. QA 做全流程回归 + 兼容性巡检。

### 协作与交接
- 地图与事件采用版本号锁定，变更需同步 QA 重跑重点用例。

## M4 Polish and optimization
### 阶段目标
- 提升手感、可读性与稳定性，达到可对外试玩质量。

### 必须完成
1. 战斗反馈（受击、击杀、镜头震动、音效）完整。
2. UI/UX 收口：建造提示、波次预警、失败原因说明。
3. 性能与加载优化、崩溃修复、日志可追踪。

### 验收标准
1. 关键场景 FPS/内存达到目标阈值。
2. 高优先级缺陷清零，中优先级可控。
3. 可交付试玩包 + 已知问题列表。

### 角色推进顺序
1. Code 优先性能与稳定性。
2. Art 完成表现层统一风格。
3. QA 做发布前回归与压测。

### 协作与交接
- 冻结内容后只允许修复性改动；任何资源替换必须触发冒烟回归。

## 依赖与交接图
1. Art 依赖 Code 的资源规格与挂点清单。
2. QA 依赖 Code 的可运行包与变更说明。
3. Code 依赖 Art 的命名规范和资源版本一致性。
4. 交接链：Code 规格 -> Art 资源 -> Code 集成包 -> QA 测试报告 -> Code 修复。
