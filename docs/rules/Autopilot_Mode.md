# 自动驾驶模式（默认执行）

## 目标

减少来回确认次数。除非触发阻塞或关键决策点，否则由代理自动推进全流程。

## 执行策略

1. 默认自动执行
- 需求拆解、任务分发、代码生成、资产清单填充、集成、冒烟测试全部自动进行。

2. 只在 Gate 汇报
- 非 Gate 阶段不向用户请求步骤性确认。
- 仅在关键验收节点提交成果并请求一次性反馈。

3. 明确阻塞才中断
- 仅当缺失必要输入、外部权限不足、或存在高风险冲突时暂停并询问。

## Gate 定义

1. Gate A: 需求冻结
- 产物: `spec/feature_input.json`, `spec/AGENT_OWNERSHIP.md`
- 产物(新增): `spec/WORKLOAD_CHECK.md`
- 检查点: 范围、工期、排除项、量化分工是否明确

2. Gate B: 核心切片完成
- 产物: `tasks/code` 的 One-Day Cutline 对应实现清单
- 检查点: 主循环可跑通

3. Gate C: 资产映射完成
- 产物: `assets/manifest/ASSET_MANIFEST.md`
- 检查点: 任务与资产一一可追踪

4. Gate D: 集成与冒烟
- 产物: 集成结果与冒烟结论
- 检查点: 是否进入人工测试收尾

## 汇报格式（每个 Gate）

1. 本 Gate 完成内容
2. 验收证据（文件路径/关键结果）
3. 风险与剩余问题
4. 下一 Gate 自动计划

## Gate A 量化强制项

1. 必须给出三方工时占比（你/帮手A/帮手B）
2. 工时占比记录到 `WORKLOAD_CHECK.md`
3. 对外协作文档只展示任务与验收，不展示个人占比策略

## 默认决策权

1. 技术实现细节默认由代理决策
2. 目录与命名规则默认按既有规范执行
3. 非功能性优化不阻塞主流程
