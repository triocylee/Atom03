# 游戏复刻通用技术拆解模板（自动派发版）

## 0. 目标

本模板用于在拿到任意游戏名后，快速完成以下动作：
1. 结构化采集游戏特征。
2. 自动判定模块启用状态（必做/可选/不适用）。
3. 自动展开三角色任务（美术/代码/测试）到 L2/L3 粒度。
4. 自动估算工作量并排期。
5. 按依赖关系并行派发，按里程碑拼合验收。

---

## 2. 特征到模块映射矩阵（自动判定规则）

### 2.1 模块列表
`M01-项目骨架` `M02-输入控制` `M03-单位框架` `M04-移动导航` `M05-交互` `M06-战斗执行` `M07-AI` `M08-关卡流程` `M09-目标规则` `M10-数值配置` `M11-UI菜单` `M12-动画` `M13-特效反馈` `M14-音频` `M15-摄像机` `M16-物理碰撞` `M17-存档进度` `M18-网络联机` `M19-性能稳定` `M20-构建可观测`

### 2.2 默认启用
- 固定必做：`M01 M02 M03 M08 M10 M11 M19 M20`
- 固定可选：`M05 M09 M14 M17`
- 条件启用：`M04 M06 M07 M12 M13 M15 M16 M18`

### 2.3 条件规则（可程序化）
1. `F09=无战斗` -> `M06=不适用` `M13=可选`
2. `F10=无AI` -> `M07=不适用`
3. `F14=无联机` 或 `F15=1` -> `M18=不适用`
4. `F05 包含 3D` -> `M15=必做` `M16=必做`
5. `F05 为 2D` -> `M15=可选` `M16=可选`
6. `F17=高` 或 `F09=高度动作` -> `M12=必做` `M13=必做`
7. `F12 包含 局外成长` -> `M17=必做`
8. `F11=程序生成` -> `M08=必做` 且 `M10`优先级至少`P0`
9. `F06>=51` 或 `F19=严格` -> `M19`优先级强制`P0`
10. `F16=高` -> `M14`优先级至少`P1`
11. `F08=高` -> `M05`优先级至少`P1`
12. `F20=高保真或近似像素级` -> 所有已启用模块优先级上调一级（`P2->P1` `P1->P0`）

### 2.4 优先级打分公式
- 基础分：`必做=70` `可选=40` `不适用=0`
- 复杂度加分：`实时+10` `3D+10` `联机+20` `高动作+15` `单位规模>50 +15`
- 精度加分：`核心体验+0` `高保真+10` `近似像素级+20`
- 映射：`>=90 => P0`，`60-89 => P1`，`1-59 => P2`，`0 => N/A`

## 3. 模块子任务字典（L2/L3）

说明：每个启用模块都必须展开到 L2，再拆三角色 L3 工单。

### 3.1 通用 L2 分层（所有模块通用）
1. `L2-A 规范与接口`：输入输出、事件、命名、目录。
2. `L2-B 核心实现`：最小可运行逻辑。
3. `L2-C 数据与资源接入`：配置表、资源映射、占位替换。
4. `L2-D 集成与回归`：跨模块联调、回归验证。
5. `L2-E 优化与发布`：性能、稳定性、构建产物。

### 3.2 三角色 L3 工单模板（每个 L2 必须具备）
- 美术 L3：`资源规格` `占位资源` `正式资源` `替换验证` `缺陷修复`
- 代码 L3：`接口定义` `功能实现` `日志埋点` `异常处理` `自动化脚本/工具`
- 测试 L3：`用例设计` `冒烟执行` `功能回归` `边界测试` `缺陷关闭验证`

### 3.3 模块特有 L2 补充项（用于细化工单）
- `M02 输入控制`：设备映射、按键冲突策略、输入缓存策略。
- `M03 单位框架`：生命周期、状态切换协议、属性读写协议。
- `M04 移动导航`：位移模型、路径更新频率、阻挡体规则。
- `M05 交互`：交互优先级、可达性判定、中断恢复。
- `M06 战斗执行`：命中判定层、伤害公式层、冷却资源层。
- `M07 AI`：感知层、决策层、行为执行层、脱战层。
- `M08 关卡流程`：流程状态机、触发器图、重开与重置。
- `M09 规则系统`：目标追踪、胜负判定、奖励结算。
- `M10 数值配置`：配置格式、校验器、热更新策略。
- `M11 UI菜单`：页面流转图、状态绑定、分辨率适配。
- `M12 动画`：状态图、过渡条件、事件点协议。
- `M13 特效反馈`：事件绑定表、播放生命周期、回收池。
- `M14 音频`：事件总线、混音分组、并发优先级。
- `M15 摄像机`：跟随策略、边界限制、遮挡处理。
- `M16 物理碰撞`：层矩阵、触发器规则、高速检测。
- `M17 存档进度`：存档结构、版本迁移、容错恢复。
- `M18 网络联机`：同步粒度、权威端策略、断线恢复。
- `M19 性能稳定`：性能预算、采样点、压力场景。
- `M20 构建可观测`：构建脚本、版本号、日志/崩溃上报。

## 4. 工作量估算与排期规则

### 4.1 复杂度分级
- `C1`（简单）：已有现成模式，改造少。
- `C2`（中等）：需模块内联动。
- `C3`（复杂）：跨模块联动或关键路径。
- `C4`（高复杂）：高风险、需反复调优。

### 4.2 复杂度到人天换算（基准）
- 美术：`C1=0.5` `C2=1.0` `C3=2.0` `C4=3.0`
- 代码：`C1=0.5` `C2=1.5` `C3=3.0` `C4=5.0`
- 测试：`C1=0.5` `C2=1.0` `C3=1.5` `C4=2.5`

### 4.3 修正系数
- 3D 项目：`x1.2`
- 联机项目：`x1.4`
- 高保真：`x1.25`
- 严格性能预算：`x1.2`
- Jam 短周期（<=7天）：仅允许保留 `P0/P1`，`P2`自动延后。

### 4.4 自动派发规则
1. 先派所有 `P0` 到最近里程碑。
2. 同角色每日容量上限：`1 人 = 1.0 人天/天`。
3. 单任务估算 > 2 人天时，强制拆分为两个及以上 L3 工单。
4. 任意任务无前置时可并行派发；有前置时进入等待队列。
5. 若里程碑前 `P0` 完成率 < 85%，自动冻结 `P1` 新增任务。

## 5. 依赖与并行规则

### 5.1 标准前置关系
1. `M01 -> 全模块`
2. `M02 -> M03 M04 M06`
3. `M03 -> M06 M07 M12`
4. `M10 -> M06 M07 M09 M11 M17`
5. `M12(事件协议) -> M06 M13 M14`
6. `M08 -> M09 M11`
7. `M16 -> M04 M06`
8. `M18 -> M09 M11 M17`
9. `M19`与`M20`全程并行，但发布前必须最终通过。

### 5.2 可并行条件
- 只要接口协议冻结（字段、事件ID、回调签名不变），资源制作可与代码实现并行。
- 测试可在“占位可运行包”阶段提前执行冒烟和回归框架搭建。

### 5.3 阻塞升级规则
- 阻塞 > 4 小时：模块负责人升级到项目负责人。
- 阻塞 > 1 天：触发降级策略（保留 P0，裁剪 P1/P2）。
- 接口变更：必须同步更新 `接口变更记录`，并触发受影响模块重估算。

## 7. 里程碑验收包标准

### M1 骨架可运行
- 必交：可运行包、模块启用清单、冒烟报告。

### M2 垂直切片
- 必交：完整流程录屏、P0任务完成证明、主要缺陷列表。

### M3 内容拼合
- 必交：P1任务收敛报告、回归报告、性能对比报告。

### M4 发布候选
- 必交：最终构建包、全量验收报告、已知风险清单。

## 8. 执行顺序（拿到游戏名后的标准流程）

1. 填《特征采集表》（第1章）。
2. 跑《映射矩阵规则》（第2章）生成模块清单。
3. 按《子任务字典》（第3章）展开 L2/L3 工单。
4. 用《估算规则》（第4章）计算人天并排期。
5. 用《依赖规则》（第5章）调整并行与前置。
6. 输出标准 JSON/CSV（第6章），进入执行。
7. 按里程碑提交验收包（第7章）。
